# RAVA_RNG

The `RAVA_RNG` library implements the functionality shared by all hardware implementation of the
RAVA circuit architecture.

The architecture assumes the following hardware components:
- A PWM signal driving a boost stage that generates the elevated voltage applied to the
  reverse-biased Zener diodes
- Similar avalanche noise sources whose signals are continuously compared
- A comparator output producing a digital signal indicating which avalanche source has the greater
  instantaneous amplitude
- A counter peripheral used to count rising-edge pulses generated by the comparator during a
  configurable sampling interval
- A communication interface for message exchange between the RAVA device and the host system

The library provides the following functionality:
- A framed binary communication protocol, including methods for receiving and transmitting protocol
  messages
- Incoming messages are processed using a byte-oriented state machine parser
- Command-dispatch functionality for decoding received commands and invoking their associated
  handler functions
- Health-monitoring methods compliant with the NIST SP 800-90B recommendations, including:
  - Startup validation tests
  - Continuous runtime entropy tests
- Generic methods, for generating and transmitting:
  - Pulse counts
  - Random bits
  - Random bytes
  - Uniformly distributed integers
  - Uniformly distributed floating-point values
- General-purpose utilities and helper functions used throughout the framework

Implementation applications are responsible for providing hooks for device, PWM boost, and RNG
functionality.

Importantly, the implementation must define the `rng_gen_byte()` function, which applies the
underlying hardware mechanisms to extract the parity of the pulse counts, generate random bits, and
assemble them into random bytes.

Additionally, applications must implement communication interfaces for message transport, providing
the methods defined in the `comm_interface_t`.
